# Chorus

AI 微信群产品：**AstrBot 当瘦多 bot 适配层**（只收发字节）+ **独立 LangGraph 编排服务当大脑**（人设/调度/记忆/LLM 全在此），两者用一条窄 HTTP 桥解耦。协作模式 = 可组合的配方 DAG（圆桌/扇出/auto…），引擎只给原语。

## 接手本仓库：先读热启动盘

**第一步永远是读根目录 `SESSION_CHECKPOINT.md`** —— 它是入口与热内存，告诉你「当前在做什么 · 下一步 · 哪些未提交」，并带一份冷启动读序路由到架构文档。它覆盖式重写、可能过期，按其顶部「新鲜度自检」对比 `git log` 后再信。

## 持久文档（架构与账本，单一真相源）

- `docs/技术方案-基于AstrBot的MVP实现.md` — 架构骨架 + §6 决策档（按编号查来龙去脉）
- `docs/代码链路.md` — 文件级改动账本，刀刀追加（精确到 `文件:符号`）
- `docs/切片计划-基于AstrBot的MVP.md` — 切片清单与进度（✅/🅿️）

## 工作约定

- **测试**：orchestrator 用 `orchestrator/.venv` 跑 pytest（系统 Python 3.14 缺依赖）。
- **提交**：每切片完成直接 commit + push main，无需问。
- **文档纪律**：每刀完成在 `代码链路.md` 追加一条；收口/上下文将断时刷新 `SESSION_CHECKPOINT.md`（整页覆写）。
