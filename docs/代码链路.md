## 2026-06-01 · 流式输出（SSE）—— 解决"输出慢/超时"（学自 AstrBot）

**根因**：原 `/inbound` 非流式等全部生成完才返回；kimi-k2.6 等 reasoning 模型长输出 → >180s → axios 超时；且非流式还会空返回/连接重置。**cap 输出不可行**（cap 太小把 reasoning 的 JSON 挤空）。
**学自 AstrBot**（`dashboard/routes/chat.py`）：SSE `text/event-stream` + 队列 `get(timeout=1)` 超时发心跳 `: heartbeat\n\n` 保活 + `response.timeout=None` + typed events。总时间不变，但首字节快、token 流出、无超时墙。

**触达**：
- `orchestrator/app/llm.py:robust_ainvoke` — `ainvoke`→`astream` 累积（修 kimi 非流式空返回）+ `config` 透传
- `orchestrator/app/nodes/generate.py` — 每 agent 的 LLM 调用打 `tags=["agent:<id>"]`，供 LangGraph messages 流路由
- `orchestrator/app/service.py:inbound_stream` — `POST /inbound/stream` SSE：`graph.astream(stream_mode=["updates","messages"])` + 队列心跳；事件 `status`/`framed`/`delta`/`candidates`/`done`/`error`
- `orchestrator/app/config.py` — 可选 `LLM_MAX_TOKENS`（默认 None=不限）
- `orchestrator/tests/test_stream.py`；模型假桩 astream 加 config 形参

**入口**：`POST /inbound/stream`（SSE，前端用）；`/inbound` 非流式保留（测试/兼容）。
**测试**：`pytest` 29 passed（SSE 事件序列 framed/candidates/done + 流式后 /curate 可用）。**spike 实测**：fanout token chunk 带 `tags=["agent:B"]` → 按 agent 路由 OK；首个 fanout token ~16s（frame 后），心跳保活；空 content chunk（reasoning）跳过。
**注**：Model A 保持——仍是 `graph.astream` 跑配方图，SSE 只转发。

**前端（本机验）**：
- `web/src/api/chorus.js:inboundStream` — fetch + ReadableStream 手解析 SSE（POST 故不用原生 EventSource），按事件 type 分派
- `web/src/views/CuratePage.vue` — `runInbound` 改流式：`status`→显示"分配维度中…"；`framed`→建空候选卡；`delta`→按 contact_id 逐 token 追加到对应卡；`candidates`→最终权威版
- 判据（浏览器）：长问题不再超时；维度出现后 token 边生成边逐卡冒出（N 人同时打字）。
- **Markdown 渲染**（移植 AstrBot 外壳件）：`web/src/utils/markdown.js`（`markdown-it` + `dompurify` 防 XSS，html:false）；CuratePage 候选卡 + 产出用 `v-html="renderMd()"`（scoped `:deep()` 样式）。流式 delta 增长时实时重渲染。需 `npm install`（新增 markdown-it/dompurify）。

## 2026-06-02 · S3.0 配方引擎 interrupt 化（人在环统一，模型 A，§6.10）

**触达**：

## 2026-06-02 · S3.6d 圆桌续场 + 插话异步注入端点

**动因**：S3.6c 只能起场（停在 human_gate）。本切片补两条续场通道——同步 resume（当场插话/继续/答澄清）+ 异步 interject（外部写 pending_human，下次消化），圆桌人在环闭环。

- `orchestrator/app/service.py:roundtable_interject` — `POST /roundtable/{key}/interject`：`aupdate_state` 写 `pending_human`（异步通道），下次 human_gate resume 时消化（改向）
- `orchestrator/app/service.py` — 加 `RoundtableResumeReq`/`InterjectReq` 模型
- `orchestrator/tests/test_roundtable_service.py` — 加 4 测试

**入口**：`POST /roundtable/{key}/resume/stream`（续/插话/答澄清）、`POST /roundtable/{key}/interject`（异步注入）。
