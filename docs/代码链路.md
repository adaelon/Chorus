## 2026-06-01 · 流式输出（SSE）—— 解决"输出慢/超时"（学自 AstrBot）

**根因**：原 `/inbound` 非流式等全部生成完才返回；kimi-k2.6 等 reasoning 模型长输出 → >180s → axios 超时；且非流式还会空返回/连接重置。**cap 输出不可行**（cap 太小把 reasoning 的 JSON 挤空）。
**学自 AstrBot**（`dashboard/routes/chat.py`）：SSE `text/event-stream` + 队列 `get(timeout=1)` 超时发心跳 `: heartbeat\n\n` 保活 + `response.timeout=None` + typed events。总时间不变，但首字节快、token 流出、无超时墙。

**触达**：
- `orchestrator/app/llm.py:robust_ainvoke` — `ainvoke`→`astream` 累积（修 kimi 非流式空返回）+ `config` 透传
- `orchestrator/app/nodes/generate.py` — 每 agent 的 LLM 调用打 `tags=["agent:<id>"]`，供 LangGraph messages 流路由
- `orchestrator/app/service.py:inbound_stream` — `POST /inbound/stream` SSE：`graph.astream(stream_mode=["updates","messages"])` + 队列心跳；事件 `status`/`framed`/`delta`/`candidates`/`done`/`error`
- `orchestrator/app/config.py` — 可选 `LLM_MAX_TOKENS`（默认 None=不限）
- `orchestrator/tests/test_stream.py`；模型假桩 astream 加 config 形参

**入口**：`POST /inbound/stream`（SSE，前端用）；`/inbound` 非流式保留（测试/兼容）。
**测试**：`pytest` 29 passed（SSE 事件序列 framed/candidates/done + 流式后 /curate 可用）。**spike 实测**：fanout token chunk 带 `tags=["agent:B"]` → 按 agent 路由 OK；首个 fanout token ~16s（frame 后），心跳保活；空 content chunk（reasoning）跳过。
**注**：Model A 保持——仍是 `graph.astream` 跑配方图，SSE 只转发。前端 CuratePage 改流式消费 = 下一步（本机验）。
